# Contributing to Shipyard

Thank you for your interest in contributing to Shipyard! This document provides guidelines for contributing to the project.

## How to Contribute

### Reporting Issues

If you find a bug or have a feature request, please open an issue on GitHub. Include as much detail as possible:

1. What you expected to happen
2. What actually happened
3. Steps to reproduce the issue
4. Your environment (OS, Claude Code version, etc.)

### Submitting Changes

1. Fork the repository
2. Create a feature branch from `main`
3. Make your changes
4. Write clear commit messages explaining the "why" behind changes
5. Open a Pull Request with a description of your changes

### Code Style

Since Shipyard is primarily markdown specifications and Claude Code skills, please follow these guidelines:

**Markdown**: Use consistent heading levels, include code examples where helpful, and keep lines readable.

**Skills**: Follow the patterns established in `docs/skills/` and align with Claude Code skill best practices.

**Scripts**: If adding bash or Python scripts, include comments explaining non obvious logic.

### Documentation

All features should be documented. If you add a new capability, update the relevant docs in the `docs/` folder.

## Questions?

Open a Discussion on GitHub if you have questions about contributing.
